# Devpost Draft

## One-line description

Museion turns a trusted source into inspectable, deterministic learning interactions and records exactly what a learner applies without assistance.

## What it does

Museion normalizes a source into stable pages and hashes, grounds concepts and claims in exact spans, plans a typed course, and renders interactions whose truth is owned by deterministic reducers. Maia can coach only through a strict, leak-gated boundary. A final one-attempt near-transfer task disables Maia, hints, and solution reveal, then produces a reconciled evidence statement with explicit limitations.

## How we used OpenAI

The live compiler and tutor providers target GPT-5.6 through the Responses API with strict Structured Outputs, bounded inputs/outputs, telemetry, critic validation, and at most one typed repair. The judged golden path is also checked in as a visibly labelled replay, so it remains reliable without an API key. No live-model result is claimed in the local verification reported here.

## Measured local results

- 130 offline tests passed; 8 live-model red-team cases were skipped without credentials.
- Production build, lint, TypeScript, and dependency audit passed; audit reported zero vulnerabilities.
- The complete judge path passed 20 independent desktop runs and one 320 px run with no page/console/5xx errors or horizontal overflow.
- One real six-page PDF regenerates the checked Source Document and exact citation graph deterministically.

## Claim boundary

The demo records one immediate near-transfer observation. It does not prove durable mastery, far transfer, or learning efficacy. Hosted deployment and final submission must be verified separately.
