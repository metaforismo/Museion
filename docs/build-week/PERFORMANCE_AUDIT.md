# Frontend Performance Audit

Audit date: 2026-07-15

Target: local optimized production build generated by Next.js 16.2.10

## Outcome

The audited routes score 91–100 for Lighthouse performance and 100 for accessibility, best practices, and SEO. One Judge mobile remediation run moved performance from 91 to 96 and LCP from 3.5 seconds to 2.7 seconds; the final independent run scored 92 with a 3.3-second LCP, demonstrating normal synthetic variance. The stable result is that the visible-label mismatch is removed and CLS falls to zero in both Lighthouse and the repeatable Playwright budget.

These are throttled local measurements, not field data. Scores and timings vary by machine and Lighthouse version, so the repository gates stable byte and layout budgets rather than a fragile exact Lighthouse score.

## Bundle inventory

- `.next/static`: 2,780.7 KiB of generated files, budget 3,072 KiB.
- Largest JavaScript chunk: 492.3 KiB uncompressed on disk, budget 550 KiB.
- PDF worker: 1,274.3 KiB, budget 1,400 KiB.
- The PDF parser and worker are loaded only when a learner selects a PDF. The Create initial transfer remains 284 KiB, so the worker is not part of ordinary route startup.

## Enforced budgets

`npm run verify:bundle` runs after the production build in CI and fails when:

- total generated static assets exceed 3 MiB;
- the largest JavaScript chunk exceeds 550 KiB;
- the PDF worker exceeds 1,400 KiB or disappears unexpectedly.

`npm run verify:ui` opens fresh mobile browser contexts for Welcome, Create, and Judge and fails when:

- initial encoded transfer exceeds 400 KiB;
- JavaScript transfer exceeds 250 KiB;
- cumulative layout shift exceeds 0.02.

## Changes made from the audit

1. Removed a redundant home-link `aria-label` that did not contain all visible label text; the decorative monogram is now hidden from assistive technology.
2. Reserved stable vertical space while the Judge creates its replay session so the footer does not jump through the viewport.
3. Added route-level transfer and CLS measurements to the existing real-browser verification.
4. Added deterministic static bundle budgets to local verification and GitHub Actions.

## Honest boundary

Local Lighthouse cannot establish production Core Web Vitals, CDN behavior, real network latency, or field interaction latency. Hosted verification should be repeated after an authorized deployment, and real-user monitoring should be considered only with an explicit privacy and retention policy.
